# Descripción General

Crea y actualiza menús utilizando los puntos finales de la **API V3 de Listas**. Cada tienda puede tener su propio menú configurable de manera individual.

<Info>
  **Producto**

  Se muestra como cualquier artículo que un usuario puede seleccionar del menú de una tienda, incluyendo postres, bebidas y condimentos.
</Info>

<Info>
  **Categoría**

  Agrupa uno o más productos, como complementos, platos principales y bebidas.
</Info>

<Info>
  **Grupos de Modificadores**

  Agrupa modificadores que pueden ser seleccionados como personalizaciones bajo un artículo principal. Por ejemplo, el grupo de modificadores "Sabor de Bebida" contiene opciones como fresa, limón y naranja.
</Info>

<Info>
  **Catálogo (Canal)**

  Agrupa una o más categorías de una tienda. Permite hacer referencia a dos tipos de cumplimiento: Recogida y Entrega.
</Info>

![Diagrama API Menú](https://s3.us-east-1.amazonaws.com/admin.kfc.production/menuintegration.jpeg)

# Componentes

- **Obtener Reporte de Sincronización (GET)**
Este punto final proporciona un reporte detallado de la sincronización de datos, permitiéndote monitorear y verificar el estado de las sincronizaciones realizadas.

- **Registrar o Editar una Lista (PUT)**
Este punto final se usa para agregar nuevas tiendas y canales de venta al sistema. Facilita la expansión eficiente y la actualización de la red de tiendas.

# Preguntas Frecuentes

**¿Qué especificaciones deben cumplir las imágenes de los elementos del menú para un procesamiento exitoso?**

Las imágenes deben tener un tamaño de 744px x 744px.

**¿Puedo establecer horarios de disponibilidad específicos para categorías, elementos o modificadores?**

Sí, es posible establecer horarios de disponibilidad tanto para **categorías** como para **elementos**. Debes usar el atributo _“schedules”:[ ]_ para ambos casos. En el caso de **modificadores**, no es posible asignar un horario.
