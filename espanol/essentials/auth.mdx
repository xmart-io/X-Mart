## Requisitos

El acceso a esta API puede requerir aprobación y configuración previas. Se recomienda revisar las guías de inicio rápido antes de la integración.

## Parámetros de la solicitud

Las solicitudes deben enviarse en formato application/json e incluir los siguientes parámetros:

| **Key**         | **Type**   | **Description** |
|----------------|-----------|----------------|
| `client_id`    | `string`  | Identificador único para la aplicación cliente. |
| `client_secret` | `string`  | Clave secreta de la aplicación cliente usada para autenticación. |
| `grant_type`   | `string`  | Tipo de concesión usado para la autenticación. |

## Ejemplo de Solicitud

<CodeGroup>

```bash Example request
curl --request POST \
  --url https://api.artisn.desarrollo-redbrand.com/api/authentication/login \
  --header 'Content-Type: application/json' \
  --data '{
    "client_id": "<string>",
    "client_secret": "<string>",
    "grant_type": "<string>"
}'
```
</CodeGroup>

| **Key**        | **Type**   | **Description** |
|---------------|-----------|----------------|
| `accessToken` | `string`  | Token de acceso generado, utilizado para autenticarse contra los endpoints protegidos. |

## Usando el Token

El accessToken obtenido debe enviarse en el encabezado de la solicitud al llamar a los endpoints protegidos.
